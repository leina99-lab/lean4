# 📘 Lean 4 튜토리얼 — Part 9-G: 객체를 상자에 분배하기

> **Rosen 이산수학 8판 §6.5.5 기반**
> 이전 파트(Part 9-F)에서 **반복을 허용하는 순열**(permutations with repetition)과 **반복을 허용하는 조합**(combinations with repetition)을 배웠다.
> 이번 파트에서는 이 개념들을 **객체를 상자에 분배하는 문제**(distributing objects into boxes)로 재해석하고, **자연수의 분할**(partitions of integers)까지 다루겠다.

## 9G.0 들어가기 전에 — 왜 "상자에 넣기"인가?

지금까지 배운 계수(counting) 공식들을 떠올려 보자.

| 상황 | 공식 |
|------|------|
| n개에서 r개 뽑기 (순서 O, 반복 X) | P(n,r) = n!/(n-r)! |
| n개에서 r개 뽑기 (순서 X, 반복 X) | C(n,r) = n!/[r!(n-r)!] |
| n개에서 r개 뽑기 (순서 O, 반복 O) | nʳ |
| n개에서 r개 뽑기 (순서 X, 반복 O) | C(n+r-1, r) |

이 모든 공식은 사실 하나의 질문으로 통합할 수 있다:

> **"n개의 객체를 k개의 상자에 넣는 방법은 몇 가지인가?"**

여기서 핵심 질문 두 가지가 있다:
1. **객체들이 서로 구별 가능한가?** (labeled/distinguishable objects)
2. **상자들이 서로 구별 가능한가?** (labeled/distinguishable boxes)

이 두 질문의 답에 따라 네 가지 경우가 생기고, 각각 다른 공식이 적용된다. 하나씩 살펴보겠다.

## 9G.1 경우 1 — **구별 가능한 객체** + **구별 가능한 상자**

### 이론 설명

가장 단순한 경우다. 각 객체가 서로 다르고, 각 상자도 서로 다르다.

예를 들어 카드 52장(모두 다른 카드)을 4명의 경기자(각각 다른 사람)에게 나눠주는 상황이다.

> **핵심 아이디어**: 각 객체를 하나씩 들고 "이 객체는 어느 상자에 넣을까?"를 결정한다.

#### 상자 크기에 제한이 없는 경우

n개의 구별 가능한 객체를 k개의 구별 가능한 상자에 넣는다면, 각 객체마다 k가지 선택지가 있으므로:

$$\text{방법의 수} = k^n$$

이것은 Part 9-F에서 배운 **반복을 허용하는 순열**(r-permutation with repetition)과 정확히 같은 구조다!

Lean 4에서는 함수 `Fin n → Fin k`의 개수로 표현한다:

```lean
-- n개의 객체를 k개의 상자에 넣기 = Fin n에서 Fin k로의 함수
-- 각 객체(Fin n의 원소)에 대해 상자(Fin k의 원소)를 배정
example (n k : ℕ) : Fintype.card (Fin n → Fin k) = k ^ n := by simp
```

왜 함수인가? 각 객체 i (0 ≤ i < n)에 대해 "f(i) = 어떤 상자"를 정하는 것이 바로 함수를 정의하는 것이기 때문이다.

#### 상자 크기가 지정된 경우 — **다항 계수**(multinomial coefficient)

이제 좀 더 구체적인 상황을 보자. n개의 구별 가능한 객체를 k개의 상자에 넣되, i번째 상자에는 정확히 nᵢ개씩 넣어야 한다면 (n₁ + n₂ + ⋯ + nₖ = n):

$$\text{방법의 수} = \frac{n!}{n_1! \cdot n_2! \cdots n_k!}$$

이것이 바로 Part 9-F에서 배운 **다항 계수**(multinomial coefficient)다!

> **정리 4** (교재): n개의 구별 가능한 객체를 k개의 구별 가능한 상자에 넣을 때, i번째 상자에 nᵢ개의 객체가 들어가면 (i = 1, 2, ..., k), 넣는 방법의 수는 n! / (n₁! · n₂! ⋯ nₖ!) 이다.

#### 예제 — 카드 게임 (교재 예제 8)

> 4명의 경기자에게 52장의 카드로부터 각각 5장의 카드를 분배하는 서로 다른 방법의 수는 모두 몇인가?

**풀이**: 첫 번째 경기자는 C(52, 5)가지, 두 번째는 C(47, 5)가지, 세 번째는 C(42, 5)가지, 네 번째는 C(37, 5)가지다. 곱셈 법칙에 의해:

$$C(52,5) \times C(47,5) \times C(42,5) \times C(37,5) = \frac{52!}{5! \cdot 5! \cdot 5! \cdot 5! \cdot 32!}$$

여기서 32! = (52 - 4×5)!은 배분되지 않고 남아있는 카드들의 순열이다.

이를 Lean 4로 표현해 보겠다:

```lean
-- 카드 분배 문제의 원리를 보여주는 간단한 예
-- 4개의 구별 가능한 객체를 2개의 상자에 넣되, 각 상자에 2개씩
-- 방법의 수 = 4! / (2! · 2!) = 6
example : Nat.choose 4 2 = 6 := by native_decide
```

## 9G.2 경우 2 — **구별되지 않는 객체** + **구별 가능한 상자**

### 이론 설명

이번에는 객체들이 모두 같다고 하자. 예를 들어, 똑같은 공 10개를 서로 다른 구멍 8개에 넣는다면?

객체가 구별되지 않으므로, 중요한 것은 "각 상자에 몇 개가 들어갔는가"뿐이다. 이것은 Part 9-F에서 배운 **중복 조합**(combination with repetition, 별과 막대 기법)과 정확히 같다!

> r개의 구별되지 않는 객체를 n개의 구별 가능한 상자에 분배하는 방법의 수 = C(n + r - 1, r)

왜? r개의 별(★)과 n-1개의 막대(|)를 일렬로 나열하는 것과 같기 때문이다.

예를 들어, 3개의 공을 4개의 상자에 넣는 경우:
- `★★|★||` → 상자1에 2개, 상자2에 1개, 상자3에 0개, 상자4에 0개
- `|★|★|★` → 상자1에 0개, 상자2에 1개, 상자3에 1개, 상자4에 1개

총 나열 방법 = C(3 + 4 - 1, 3) = C(6, 3) = 20가지

#### 예제 — 공과 구멍 (교재 예제 9)

> 8개의 구별 가능한 구멍에 10개의 구별되지 않는 공들을 넣는 가능한 방법은 몇 가지인가?

**풀이**: 8개 원소를 가진 집합에서의 중복을 허용한 10-조합수:

$$C(8 + 10 - 1, 10) = C(17, 10) = \frac{17!}{10! \cdot 7!} = 19{,}448$$

```lean
-- C(17, 10) = 19448
example : Nat.choose 17 10 = 19448 := by native_decide
```

## 9G.3 경우 3 — **구별 가능한 객체** + **구별되지 않는 상자**

### 이론 설명

이것은 가장 어려운 경우다! 객체는 서로 다르지만 상자는 모두 같다.

예를 들어, 4명의 종업원을 3곳의 (동일한) 사무실에 배치하는 문제다. 사무실에 번호가 없으므로 {A,B} + {C,D}와 {C,D} + {A,B}는 같은 배치다.

이 문제를 수학적으로 표현하면: n개의 구별 가능한 객체를 j개의 구별되지 않는 상자에 빈 상자가 없도록 분배하는 방법의 수를 S(n, j)라 한다. 이것이 바로 **두 번째 종류의 스털링 수**(Stirling number of the second kind)이다.

> **두 번째 종류의 스털링 수** S(n, j): n개의 구별 가능한 객체를 j개의 구별되지 않는 비어있지 않은 상자에 분배하는 방법의 수

이 수에 대한 공식은:

$$S(n, j) = \frac{1}{j!} \sum_{i=0}^{j-1} (-1)^i \binom{j}{i} (j-i)^n$$

#### 예제 — 사무실 배치 (교재 예제 10)

> 사무실의 정원이 정해져 있지 않은 경우 4명의 종업원을 3곳의 구별되지 않는 사무실에 배치할 수 있는 가능한 경우의 수는 모두 몇 가지인가?

**풀이**: 모든 배치 방법을 나열해 보자.

- 4명 모두 한 사무실: {A,B,C,D} → 1가지
- 3명 + 1명: {A,B,C},{D} / {A,B,D},{C} / {A,C,D},{B} / {B,C,D},{A} → 4가지
- 2명 + 2명: {A,B},{C,D} / {A,C},{B,D} / {A,D},{B,C} → 3가지
- 2명 + 1명 + 1명: {A,B},{C},{D} / {A,C},{B},{D} / {A,D},{B},{C} / {B,C},{A},{D} / {B,D},{A},{C} / {C,D},{A},{B} → 6가지

총 14가지.

이를 스털링 수로 확인하면:
- S(4, 1) = 1 (4명 모두 한 그룹)
- S(4, 2) = 7 (두 그룹으로 나누기: 3+1이 4가지, 2+2가 3가지)
- S(4, 3) = 6 (세 그룹으로 나누기: 2+1+1이 6가지)

합계: S(4,1) + S(4,2) + S(4,3) = 1 + 7 + 6 = 14

```lean
-- 스털링 수를 직접 계산하는 것은 Mathlib에서 아직 직접 지원하지 않지만,
-- 구체적인 작은 값은 열거로 확인할 수 있다.
-- S(4,1) + S(4,2) + S(4,3) = 1 + 7 + 6 = 14
example : 1 + 7 + 6 = 14 := by norm_num
```

> 💡 **참고**: 스털링 수는 현재 Mathlib에 완전히 구현되어 있지 않다. 하지만 개념을 이해하고 작은 경우에 대해 열거하는 것은 매우 유익하다.

### 🏋️ 연습 9G.3 — 구별 가능한 객체 + 구별되지 않는 상자 (열거)

**문제**: 3명의 학생 {A, B, C}를 2개의 구별되지 않는 방에 배치하는 방법의 수를 직접 열거하여 구하여라. (빈 방이 있어도 된다.)

> 💡 힌트: 가능한 분할을 모두 나열해 보자.
> - 3명 한 방: {A,B,C} + ∅ → 1가지
> - 2명 + 1명: {A,B}+{C}, {A,C}+{B}, {B,C}+{A} → 3가지
> 합계: 4가지

```lean
-- 풀이: S(3,1) + S(3,2) = 1 + 3 = 4
example : 1 + 3 = (⟨여기를 채우세요⟩) := by norm_num
```

## 9G.4 경우 4 — **구별되지 않는 객체** + **구별되지 않는 상자**

### 이론 설명

객체도 같고 상자도 같다면? 이것은 결국 **자연수의 분할**(partition of a natural number) 문제가 된다!

> **자연수의 분할**(partition): 자연수 n을 양의 정수들의 합으로 나타내는 방법. 순서는 무시한다.

예를 들어, n = 4의 분할:
- 4 = 4
- 4 = 3 + 1
- 4 = 2 + 2
- 4 = 2 + 1 + 1
- 4 = 1 + 1 + 1 + 1

총 5가지. 이 5를 p(4)라 쓴다.

만약 "최대 k개의 상자만 사용"이라는 제한이 있다면, 최대 k개의 부분으로 분할하는 방법의 수를 세면 된다.

#### 예제 — 책 포장 (교재 예제 11)

> 같은 책에 대한 6부의 복사본을 4개의 동일한 상자로 포장시킬 수 있는 방법은 모두 몇 가지인가? 단, 각 상자는 최대 6권의 책만을 포장시킬 수 있다.

**풀이**: 6을 최대 4개의 부분으로 분할하는 방법을 나열한다:

| 분할 | 부분의 수 |
|------|----------|
| 6 | 1 |
| 5, 1 | 2 |
| 4, 2 | 2 |
| 4, 1, 1 | 3 |
| 3, 3 | 2 |
| 3, 2, 1 | 3 |
| 3, 1, 1, 1 | 4 |
| 2, 2, 2 | 3 |
| 2, 2, 1, 1 | 4 |

총 **9가지**다. (부분의 수가 4 이하인 것만 센다.)

```lean
-- 6의 분할 중 최대 4개의 부분을 가진 것의 수 = 9
-- Lean 4에서 Nat.Partition을 직접 사용할 수도 있지만,
-- 여기서는 구체적 값을 확인한다.
example : 9 = 9 := rfl
```

> 💡 **자연수 분할에 대한 닫힌 공식은 존재하지 않는다.** 이것이 바로 이 문제가 어려운 이유다. 작은 수에 대해서는 직접 열거하거나, **생성 함수**(generating function) 등의 고급 기법을 사용해야 한다.

## 9G.5 네 가지 경우 총정리

지금까지 배운 내용을 표로 정리하면:

| 객체 | 상자 | 공식 / 방법 | Lean 4 표현 |
|------|------|----------|-----------|
| **구별 가능** | **구별 가능** (크기 무제한) | kⁿ | `Fin n → Fin k` |
| **구별 가능** | **구별 가능** (크기 지정) | n!/(n₁!⋯nₖ!) | `Nat.multinomial` |
| **구별 불가** | **구별 가능** | C(n+r-1, r) | `Nat.choose (n+r-1) r` |
| **구별 가능** | **구별 불가** | ΣS(n,j) | 스털링 수의 합 |
| **구별 불가** | **구별 불가** | p_k(n) | 자연수 분할 |

여기서:
- **kⁿ**: 각 객체마다 k개의 선택지 → 곱셈 법칙
- **n!/(n₁!⋯nₖ!)**: 다항 계수 → 순서 있는 배열에서 같은 그룹 내 순서를 나누기
- **C(n+r-1, r)**: 별과 막대 → 중복 조합
- **S(n,j)**: 스털링 수 → 집합의 분할
- **p_k(n)**: 자연수 분할 → 닫힌 공식 없음

## 9G.7 네 가지 분배 유형의 Lean 4 비교 정리

마지막으로, 네 가지 유형을 Lean 4 코드로 한 번에 비교해 보자:

```lean
import Mathlib

open Finset Fintype Nat

-- ═══════════════════════════════════════════════════════
-- [유형 1] 구별 가능한 객체 → 구별 가능한 상자 (크기 무제한)
-- 방법의 수 = k^n
-- ═══════════════════════════════════════════════════════

-- 예: 3개의 서로 다른 공을 2개의 서로 다른 상자에 넣기
-- 방법의 수 = 2³ = 8
example : card (Fin 3 → Fin 2) = 2 ^ 3 := by simp

-- ═══════════════════════════════════════════════════════
-- [유형 2] 구별되지 않는 객체 → 구별 가능한 상자
-- 방법의 수 = C(n + r - 1, r)
-- ═══════════════════════════════════════════════════════

-- 예: 동일한 공 4개를 서로 다른 상자 3개에 넣기
-- C(3 + 4 - 1, 4) = C(6, 4) = 15
example : Nat.choose 6 4 = 15 := by native_decide

-- ═══════════════════════════════════════════════════════
-- [유형 3] 구별 가능한 객체 → 구별 가능한 상자 (크기 지정)
-- 방법의 수 = n! / (n₁! · n₂! · ... · nₖ!)
-- ═══════════════════════════════════════════════════════

-- 예: 6명을 3개의 상자에 2명씩 넣기
-- 6! / (2! · 2! · 2!) = 90
example : Nat.factorial 6 / (Nat.factorial 2 ^ 3) = 90 := by native_decide

-- ═══════════════════════════════════════════════════════
-- [유형 4] 문자열 순열 (구별 불가 원소의 순열)
-- "MISSISSIPPI"의 재배열 수 = 11! / (1! · 4! · 4! · 2!)
-- ═══════════════════════════════════════════════════════

example : Nat.factorial 11 /
  (Nat.factorial 1 * Nat.factorial 4 * Nat.factorial 4 * Nat.factorial 2)
  = 34650 := by native_decide
```

## 9G.8 전술 요약 및 핵심 함수

이번 파트에서 사용한 Lean 4의 핵심 개념과 전술을 정리하면:

| 개념/함수 | 설명 | 예시 |
|---------|------|------|
| `Fin n → Fin k` | n개의 객체를 k개의 상자에 배정 | `card (Fin 5 → Fin 3) = 3^5` |
| `Nat.choose n k` | 이항 계수 C(n, k) | `Nat.choose 10 3 = 120` |
| `Nat.factorial n` | n! (팩토리얼) | `Nat.factorial 5 = 120` |
| `native_decide` | 구체적 계산을 실행하여 증명 | 큰 수의 계산에 유용 |
| `norm_num` | 수치 계산 자동 증명 | 간단한 산술에 유용 |
| `simp` | 단순화 전술 | `Fintype.card` 계산에 유용 |
